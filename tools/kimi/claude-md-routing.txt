## Delegation Tools (token saving)

You have CLIs for offloading token-heavy I/O to a configured *delegate* model —
any OpenAI-compatible endpoint (the default ships pointing at Moonshot's Kimi).
Use them to keep your context window focused on reasoning. The rules below are
mandatory.

### adlc-read — bulk reading

Use when you'd otherwise read a file longer than ~400 lines, or when answering
one question would require reading 3+ files. Usage:

```
adlc-read --paths <f1> <f2> ... --question "<q>"
```

Use the returned summary instead of reading the files yourself. Pass
`--no-warn` (or export `ADLC_DELEGATE_NO_WARN=1`) to suppress the privacy notice.

### adlc-write — boilerplate generation

Use for tests, config scaffolding, docstrings, and other repetitive patterns.
Usage:

```
adlc-write --spec "<what to generate>" --context <reference file> --target <output file>
```

Then review the generated file and edit only what needs fixing. Add `--force`
to overwrite an existing target. Pass `--no-warn` (or export
`ADLC_DELEGATE_NO_WARN=1`) to suppress the privacy notice.

(The legacy `ask-kimi` / `kimi-write` names still work as shims for the two
commands above.)

### Configuration

The delegate provider is three values: a base URL, a model name, and the *name*
of an env var holding the API key. Resolution precedence (highest first): CLI
flags (`--model`, `--base-url`) > `ADLC_DELEGATE_*` env vars > config file
(`~/.claude/adlc/config.yml`, override with `ADLC_CONFIG`) > legacy
`KIMI_MODEL`/`MOONSHOT_API_KEY` env > shipped defaults. The config file stores
`api_key_env` (a variable NAME), never a key value.

### Privacy and data governance

- Delegation **transmits the file/diff contents you pass to it** to the
  configured third-party endpoint. Whether that is permitted is the adopter's
  responsibility — confirm your company's data-handling policy before enabling.
- Delegation is **disabled by default**. It is enabled only by `enabled: true`
  in the config file, `ADLC_DELEGATE_ENABLED=1` in the environment, or an
  already-set legacy `KIMI_API_KEY`/`MOONSHOT_API_KEY` (continuity for existing
  installs). `ADLC_DISABLE_DELEGATE=1` (or legacy `ADLC_DISABLE_KIMI=1`) forces
  it off.
- Only the **basename** of each path is embedded in the corpus block — full
  local paths stay on your machine.
- Every run prints a one-line `delegate: ...` notice on stderr; silence it with
  `--no-warn` or `ADLC_DELEGATE_NO_WARN=1`.

### Documentation workflow

To update docs after a working session, don't re-read the conversation and
rewrite the docs by hand. Instead:

```
extract-chat ~/.claude/projects/<proj>/<session>.jsonl -o /tmp/chat.txt
adlc-read --paths /tmp/chat.txt <doc>.md --question "What doc updates are needed? Give exact edits."
```

Then apply the returned edits. Prefer this over re-reading the conversation and
rewriting docs directly.

### When NOT to delegate

Do **not** route to the delegate for:

- Tasks under ~2000 tokens (roughly under ~150 lines of code, or a short back-and-forth) — the delegation round-trip costs more than it saves.
- Architectural decisions.
- Debugging.
- Safety- and security-critical code.
- Anything that requires exact line numbers for editing.

**Claude = thinking. The delegate = I/O.**
